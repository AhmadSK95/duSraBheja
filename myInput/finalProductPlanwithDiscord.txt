## 2) Architecture that fits your droplet + existing barbershop system

### On your droplet (24/7)

**Services**

1. `brain-bot` (Discord bot + API)
2. `brain-worker` (background jobs: parsing, transcription, embeddings, digests)
3. `redis` (queue)
4. Postgres: either:

   * **preferred:** separate DB inside same Postgres instance: `brain_db`
   * or separate schema + separate role

**Key safety moves so barbershop stays untouched**

* Separate DB user: `brain_user` (only access to brain DB/schema)
* Connection pool limits for brain services
* Statement timeouts + job timeouts
* Separate tablespaces not required, but separate DB is cleaner

### On your Mac

**`brain-collector` daemon**

* Runs on schedule (every 2–6 hours, plus a morning run)
* Collects:

  * Git repo status + recent diffs summary
  * “projects running” snapshot (we define this as: active git repos + last modified + optional docker ps)
  * Codex/Claude logs if available locally (optional and scoped)
* Pushes summaries to server via HTTPS with device token

## 3) The agent cast (small but powerful)

You said: “product on the precipice of what AI can do.” Cool. We do a **multi-agent pipeline**, but keep it operationally simple.

**Agents**

1. **Ingestor**: handles attachments, extracts text, stores blobs
2. **Classifier**: maps to bucket + entities + confidence
3. **Clarifier**: asks smallest question when confidence low
4. **Librarian**: merges into canonical notes (Projects/People)
5. **Retriever**: answers “Ask my brain” using RAG over notes + artifacts
6. **Planner**: daily digest generation
7. **Project Tracker**: GitHub + Mac collector progress summaries
8. **Curator**: writing topics + YouTube suggestions

Implementation trick: these can be “agents” as separate prompts/tools, not separate processes. One worker service is enough.

## 4) Data model (Postgres) you’ll be happy you built

### Tables (minimum)

* `artifacts`: raw inputs (message text + file refs)
* `artifact_chunks`: extracted chunks for search/RAG
* `classifications`: bucket + confidence + entities(jsonb)
* `notes`: canonical objects (Project/Person/Idea/etc.)
* `links`: relationships between notes/artifacts (graph-lite)
* `review_queue`: clarifications pending
* `digests`: daily digest payload + feedback
* `sources`: tokens/metadata for GitHub, Google, YouTube integrations

### Storage for files

* Use **DigitalOcean Spaces** (private) or droplet disk (private).
* Store only `blob_ref`, `hash`, `mime`, `size` in Postgres.
