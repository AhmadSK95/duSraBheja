### duSraBheja — Personal AI Second Brain & MCP Server (Jan 2025 – Present)

**The Name:** "duSraBheja" means "second brain" in Hindi/Urdu — literally, the other brain.

**The Problem It Solves:**
Ahmad was drowning in scattered notes, screenshots, voice memos, browser bookmarks, and random text files. The problem isn't capturing information — it's retrieving it when you need it and knowing what you know.

**What It Actually Is:**
A Discord-based AI command center. Drop anything — text, PDFs, images, audio, handwritten notes — into a Discord #inbox channel. Five specialized AI agents automatically process everything:

1. **Classifier** (Claude Haiku) — fast, cheap categorization with confidence scores and structured JSON output. Routes uncertain items (below 0.75 confidence) to human review instead of silently miscategorizing.
2. **Librarian** (Claude Sonnet) — the smart one. Checks if new info should create a new note or intelligently merge into an existing one. Deduplication at the knowledge level.
3. **Retriever** — semantic search across everything via PostgreSQL + pgvector embeddings.
4. **Clarifier** — asks for disambiguation when things are ambiguous.
5. **Storyteller** — reconstructs how decisions evolved over time. Not just "what did I save about X" but "how did my thinking about X change."

**The MCP Server (The Killer Feature):**
duSraBheja exposes 6 tools through the FastMCP protocol. This means any MCP-compatible AI (Claude Code, Codex, other agents) can **query and write to Ahmad's brain directly**. During a coding session, Claude can look up past decisions, project context, and personal notes without Ahmad copy-pasting anything. The brain is AI-interoperable.

**Hard Problems Solved:**
- **Non-blocking async architecture:** Discord bot enqueues to ARQ (Redis-backed job queue) so the bot never blocks on LLM calls or file parsing. Everything is async-first.
- **Confidence-based triage:** The 0.75 threshold means the system admits when it doesn't know, rather than guessing wrong silently. Knowing what you don't know matters more than being right most of the time.
- **Canonical note merging:** The Librarian decides whether new information creates a new note or merges. This prevents the "500 notes about the same thing" problem.
- **Story-aware retrieval:** Not just keyword/vector search, but project-aware narrative reconstruction — understanding the timeline of decisions.
- **Cost-conscious routing:** Claude Haiku for cheap classification, Sonnet for complex reasoning. Every LLM call is audit-logged with model, tokens, and cost.

**Evidence-Based Memory System:**
Raw data gets promoted into canonical memory structures: observations, episodes, threads, entities, syntheses. This isn't a note-taking app — it's a structured knowledge graph that grows over time.

**Private Dashboard + Public Profile:**
- Private dashboard for library inspection, project state management, and secret-vault access
- Public profile site powered only by approved public facts (privacy-first design)
- Chatbot for visitors, powered only by public-approved knowledge

**Tech Stack:** Python 3.12, discord.py, FastAPI, PostgreSQL 16 + pgvector, ARQ (async Redis job queue), SQLAlchemy 2.0, Alembic, Claude API (Haiku + Sonnet), OpenAI embeddings (text-embedding-3-small), FastMCP, Docker Compose

**Deployed:** DigitalOcean droplet with separate containerized services (bot, worker, MCP server, Redis, PostgreSQL). IP: 104.131.63.231

### dataGenie — Multi-Agent Data Analytics Platform (2024 – Present)

**The Problem It Solves:**
Non-technical people shouldn't need to write SQL to explore their own data. Upload a CSV, ask a question in plain English, get answers with charts and explanations.

**How It Works:**
- **Hybrid routing** — the key architectural insight. Simple questions ("what's the average revenue?") take a fast path: one LLM call generates SQL directly. Complex multi-step questions ("why did revenue drop in Q3 and how does it correlate with marketing spend?") decompose through an **agentic ReAct loop** with 5 parallel specialist agents.
- **5 Specialist Agents:** Data Quality, Statistical Analysis, Visualization, Explanation, Verification — each with specialized tools and prompts.
- **Orchestrator** decomposes complex questions into sub-tasks, routes to specialists, and synthesizes results.
- **Multi-provider LLM support:** Claude → OpenAI → Ollama with automatic fallback chains.

**Current Status (March 2026):**
- Research & Architecture: 100% complete (6 architecture docs + 5 research docs)
- Backend Core: ~60% complete
- Data Pipeline: ~40% complete
- Frontend: 0% (not started)
- 448 tests planned, 249 currently passing

**What Makes It Special:**
- Hybrid routing avoids over-engineering simple queries while handling complex ones properly
- Data profiling runs first — before answering questions, the system understands the data itself (column-level stats, quality scores, correlations, distributions)
- Confidence scores and full reasoning transparency — users see why the system gave an answer, not just the answer
- Multi-format support planned: CSV (complete), Excel (multi-sheet), PDF tables, database connectors

**Tech Stack:**
- Backend: Python 3.12, FastAPI, DuckDB (OLAP analytics), PostgreSQL, Celery + Redis (async), Docker
- Frontend (planned): Next.js 16, React 19, TypeScript, shadcn/ui, Tailwind CSS 4, Zustand, Plotly.js
- Data: Apache Parquet (PyArrow), DuckDB analytics
- LLM: Anthropic Claude, OpenAI, Ollama (local models)

**Key Learning:** Building agentic systems is less about the LLM and more about orchestration — error recovery, sub-task dependencies, and knowing when to stop decomposing.
